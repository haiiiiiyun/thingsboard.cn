ThingsBoard

一个开源的 IoT 平台

为物联网项目提供设备管理、数据收集、处理和可视化方案

了解更多

ThingsBoard 是一个开源的物联网平台，用于数据收集、处理、可视化和设备管理。

它使用物联网行业的标准协议（MQTT、CoAP和HTTP）实现设备连接，并支持云和本地部署。 ThingsBoard 具有可扩展性、容错性和高性能，因此永远不会丢失数据。

Provision and manage devices and assets

使用丰富的服务器端 API 以安全的方式配备、监视和控制物联网实体。 定义设备、资产、客户及任何其他实体之间的关系。

Collect and visualize data

以可扩展且容错的方式收集和存储遥测数据。 使用内置或自定义的小部件以及灵活的仪表板可视化您的数据。 与客户共享仪表板。

Process and React

定义数据处理规则链。 转换和规范化您的设备数据。 对传入的遥测事件，属性更新，设备不活动和用户操作发出警报。

Microservices

使用新的微服务架构，构建ThingsBoard集群并获得最大的可扩展性和容错能力。 ThingsBoard还支持云和本地部署。




合作伙伴

实时 IoT 仪表板

ThingsBoard允许您创建丰富的IoT仪表板，以实时进行数据可视化和远程设备控制。 超过30多种可定制的小部件，能为大多数IoT用例构建最终用户所有的定制仪表板。

以可靠且容错的方式从智能表获取并存储数据， 在可定制的最终用户仪表板上可视化实时和历史能耗数据， 与第三方分析框架和解决方案集成。

智慧计量

以可靠且容错的方式从智能表获取并存储数据， 在可定制的最终用户仪表板上可视化实时和历史能耗数据， 与第三方分析框架和解决方案集成。

智慧能源

通过物联网传感器收集农业生产的重要指标，例如天气状况 或设施状态，并使用由ThingsBoard平台提供的最终用户可定制的仪表板将其可视化。

智慧农场

ThingsBoard平台可通过各种传感器跟踪车辆状态并发出警报， 使用可定制的高质量小部件和仪表板 实时绘制车辆路线并同时浏览其传感器读取历史记录。

ThingsBoard 功能

遥测数据收集

以可靠的方式收集和存储遥测数据，以应对网络和硬件故障。 使用可定制的Web仪表板或服务器端API访问收集的数据。

多租户

内置支持多租户安装。单个租户可具有多个租户管理员以及数百万个设备和客户。

数据可视化

提供30多个开箱即用的小部件，并能够使用内置编辑器创建自己的小部件。 内置线图，数字和模拟仪表，地图等等。

横向可扩展性

若以群集模式添加新的Thingsboard服务器，受支持的服务器端请求数和设备数也将呈线性增加。 不会出现停机，服务器重启或应用程序错误。

IoT 规则引擎

根据实体属性或消息内容，使用灵活的规则链处理传入的设备数据。 使用自定义逻辑将数据转发到外部系统或触发警报。 在警报上配置复杂的通知链。 使用高度可定制的规则来增强服务器端功能或操作设备。 使用规则链拖放设计器定义您的应用逻辑。

容错

所有Thingsboard服务器都是相同的。没有主从关系。节点故障自动检测。 可以在不停机的情况下更换故障节点。使用可靠的NoSQL数据库保存持久数据。

设备管理

提供注册和管理 设备的功能. 允许监测客户端和配置服务端 设备属性。 为服务端应用提供 API 向设备发送RPC 命令 或反之。

安全

支持MQTT和HTTP协议的加密传输。支持设备认证和设备凭证管理。

资产管理

提供注册和管理 资产的功能。允许配置服务端 资产属性 及监测相关警报。 使用关系建立实体层次结构的能力。.

定制与整合

使用可自定义的规则链, 小部件和传输协议来扩展默认平台功能。 除了支持MQTT，CoAP和HTTP之外，ThingsBoard用户还可实现自己的传输协议或定制现有协议的行为。

警报管理

提供创建和管理与您的实体（设备，资产，客户等）相关的警报的功能。 允许实时警报监控以及将警报传播到相关实体层次结构。 在设备断开或不活动时发出警报。

100% 开源

ThingsBoard使用Apache License 2.0许可，因此您可以在商业产品中免费使用它。 您甚至可以将其托管为SaaS或PaaS服务。

微服务架构还是单一架构

支持在小型应用环境中进行单一部署。 支持升级到微服务模式，以实现高可用性和水平可伸缩性。

SQL，NoSQL和混合数据库

支持各种数据库，支持将主要实体以及遥测数据分别存储在不同数据库。
